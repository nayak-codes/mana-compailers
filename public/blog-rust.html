Rust Memory Safety & Ownership

Published: June 2026 • Reading Time: 9 mins

Rust is a modern systems programming language that focuses heavily on safety, speed, and concurrency. Developed by Graydon Hoare at Mozilla Research, Rust compiles directly to native machine code, achieving comparable speed to C and C++. Crucially, Rust guarantees memory safety without using a garbage collector. It enforces strict ownership, borrowing, and lifetime checks during compilation, preventing common memory leaks, buffer overflows, and null-pointer exceptions.

1. The Basic Hello World in Rust

Every executable Rust program contains a main function. Printing to console uses the println! macro, denoted by an exclamation mark.

fn main() {
    println!("Hello, World from Rust!");
}

2. Variables and Mutability

By default, all variables in Rust are immutable. To declare variables that can change value, you must use the mut keyword.

fn main() {
    let x = 5; // Immutable
    // x = 6;  // This would fail to compile!
    
    let mut y = 10; // Mutable
    y = 15;
    println!("y is: {}", y);
}

3. Conditionals (if-else)

Rust conditionals do not enclose the expression in parenthesis, and the conditional block must evaluate to a boolean type.

let number = 7;

if number < 5 {
    println!("Condition met.");
} else {
    println!("Condition not met.");
}

4. Loops (loop, while, for)

Rust offers three loop constructs: loop (infinite loop until explicit break), while, and for (to loop through ranges and iterators).

// For loop through a range
for i in 1..=5 {
    println!("Value: {}", i);
}

// Conditional while loop
let mut count = 3;
while count > 0 {
    println!("Count: {}", count);
    count -= 1;
}

5. The Concept of Ownership

Ownership is Rust's most unique feature. It ensures memory is managed safely. The compiler checks these rules at compile time:

  • Each value in Rust has a variable called its owner.
  • There can only be one owner at a time.
  • When the owner goes out of scope, the value is dropped (freed from memory).
fn main() {
    let s1 = String::from("hello");
    let s2 = s1; // Ownership is MOVED to s2. s1 is no longer valid!
    
    // println!("{}", s1); // This would fail to compile!
    println!("{}", s2);   // This works fine.
}

6. Borrowing and References

If you don't want to transfer ownership, you can **borrow** values by passing references (denoted by an ampersand &).

fn calculate_length(s: &String) -> usize {
    s.len()
}

fn main() {
    let s1 = String::from("hello");
    let len = calculate_length(&s1); // Borrowing s1
    println!("Length of '{}' is {}.", s1, len); // s1 remains valid!
}
